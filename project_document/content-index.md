## 目录层级

```
赛道首页
└── 主题模块（4 个）
    └── 内容单元（每个模块 2-3 组）
        └── 单篇图文（封面 + 3-5 张内容图 + 精简正文）
```

每篇内容统一回答四个问题：

| 模块 | 说明 |
|---|---|
| 为什么重要 | 这个知识点在真实学习、项目或面试里解决什么问题 |
| 核心框架 | 用 3-5 个关键词搭出理解地图 |
| 常见误区 | 初学者最容易背错、用错、想偏的地方 |
| 实践建议 | 给一个能立刻执行的学习、排查、编码或表达动作 |

## LLM 目录

LLM 目录不做“概念堆叠”，而是围绕“模型如何工作 → RAG 怎么落地 → Agent 怎么执行任务 → AI 应用如何工程化”展开。它服务两类人：想入门 AI 应用开发的人，以及想把 AI 能力做成产品/服务的人。

### LLM-01 大模型基础

目标：理解模型生成、上下文、提示词和模型选择的基本逻辑。

| ID | 标题 | 状态 | 图文钩子 |
|---|---|---|---|
| llm-1 | 大模型到底在生成什么 | 正文已接入 | 它不是查答案，而是在续写概率最高的文本 |
| llm-2 | Token、Embedding 与上下文 | 近期制作 | 模型看到的不是文字，而是被切开的表示 |
| llm-3 | Transformer 与注意力机制 | 待制作 | 为什么模型能在长文本里抓重点 |
| llm-4 | Prompt 设计基础 | 待制作 | 好提示词不是咒语，是任务说明书 |
| llm-5 | 幻觉、上下文窗口与记忆 | 待制作 | 模型什么时候会编，什么时候会忘 |
| llm-6 | 模型选择与成本意识 | 待制作 | 不是所有任务都要用最贵的模型 |

### LLM-02 RAG 知识库

目标：把私有资料接入模型，形成可解释、可追踪、可评估的问答系统。

| ID | 标题 | 状态 | 图文钩子 |
|---|---|---|---|
| llm-7 | RAG 完整链路 | 近期制作 | 解析、切分、向量化、检索、生成一条线看懂 |
| llm-8 | 文档解析与分块 | 待制作 | 文档切不好，后面全都会歪 |
| llm-9 | Embedding 与向量检索 | 待制作 | 把语义变成可以搜索的空间 |
| llm-10 | 召回、重排与引用 | 待制作 | 找得准，比回答得漂亮更重要 |
| llm-11 | RAG 答案生成 | 待制作 | 如何让模型基于资料回答，而不是自由发挥 |
| llm-12 | RAG 评估与调优 | 待制作 | 不能只凭感觉说效果好 |
